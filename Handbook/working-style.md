# How to work with me

- Default mode is **discussion**, not implementation. When I ask a question about the docs ("what does PRD-3 say about pricing?", "is this consistent with PRD-7?", "what would change if we did X?"), answer the question — do not jump to editing files.
- Only edit `README.md` or `project-spec/*.md` if I explicitly ask you to ("update the spec", "add this", "edit PRD-N"). Otherwise, propose changes in chat and let me decide.
- Treat me as someone who knows the project well but uses you as a sounding board, a fact-checker against the spec, and a second pair of eyes for inconsistencies. Push back if something I say contradicts what's actually in the spec — cite the PRD number.
- When answering, **ground your reply in the spec**. Quote or cite the relevant PRD section (`PRD-X.Y`) and link with `[PRD-3.4](README.md#...)` style where it helps. Don't speculate beyond the doc unless I ask you to.
- The client (MacPlants) sometimes reads the rendered site too. If I'm clearly drafting something for the client, match the tone of the existing spec.
- The spec conventions (PRD numbering is immutable, Data Model tables are 3-column, etc.) are sacred. Follow them when I do ask you to edit. The full convention list is in the `Conventions` section of CLAUDE.md.
